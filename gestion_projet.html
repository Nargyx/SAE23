6. Environnements, logiciels et outils déployés

Pour matérialiser l'architecture IoT et le site web, nous avons configuré et utilisé une suite complète de technologies et de logiciels :

Infrastructure IoT & Conteneurs (Docker)

Docker
Utilisé pour isoler, déployer et interconnecter rapidement les différents services de la chaîne de traitement sans conflit de dépendances.
Mosquitto
Le broker MQTT indispensable pour gérer les files d'attente de messages (Publish/Subscribe) envoyés par les capteurs des bâtiments.
Node-RED
Outil de programmation événementielle en flux (flow-based) pour intercepter les données MQTT, les filtrer via un nœud switch et les injecter vers les bases de données.
InfluxDB
Système de gestion de base de données orienté séries temporelles (TSDB), optimisé pour l'historisation des mesures chronologiques des capteurs.
Grafana
Plateforme de visualisation qui s'appuie sur InfluxDB pour concevoir des tableaux de bord métriques interactifs (courbes, jauges) destinés à la supervision.

Environnement Serveur & Développement Web

Lubuntu (Machine Virtuelle)
Système d'exploitation GNU/Linux léger utilisé comme environnement de développement et d'hébergement pour l'ensemble de la maquette.
XAMPP
Distribution logicielle regroupant le serveur web Apache, le langage PHP et le serveur de base de données MySQL nécessaires à l'exécution de l'IHM.
MySQL & phpMyAdmin
Moteur de base de données relationnelle pour stocker les structures fixes (comptes, bâtiments, salles) et interface web de gestion graphique du schéma SQL.

Gestion de Version & Outils Collaboratifs

Git & GitHub
Outils de gestion de versions décentralisée pour suivre l'historique des modifications du code, travailler en équipe sur des branches distinctes et centraliser le projet sur un dépôt distant.
Crontab (cron)
Planificateur de tâches natif sous Linux, configuré pour exécuter automatiquement et à intervalles réguliers nos scripts PHP/Bash de collecte.

7. Conclusion et degré de satisfaction

Le projet SAÉ 23 nous a permis de concrétiser les notions théoriques acquises au cours du semestre, notamment l'interconnexion entre des infrastructures systèmes (Docker, serveurs web) et le développement de scripts ou d'interfaces dynamiques. Nos objectifs initiaux concernant la collecte, le stockage et la restitution fluide des données des capteurs de l'IUT ont été atteints.

Degré de satisfaction vis-à-vis du cahier des charges : 70%. L'ensemble des contraintes conceptuelles (modélisation de la base de données, isolation des rôles utilisateurs) et techniques (sécurisation des sessions, documentation des codes en anglais) ont été respectées.
